# AE WatchRender Manager

After Effectsの標準機能である「監視フォルダー (Watch Folder)」によるネットワーク/分散レンダリングの仕組みを補完し、レンダリングタスクを効率的かつ視覚的に管理するためのWindows向けデスクトップアプリケーションです。

## 概要

After Effectsの「ファイルを収集」によって生成されたプロジェクトフォルダと、その中に含まれる `_RCF.txt` (Render Control File) を直接監視・解析します。従来のAE標準機能では分かりにくかった「現在どのプロジェクトがレンダリング待ちか」「どれが完了・エラーになったか」を一覧で確認し、不要になった完了済みプロジェクトやログファイルを安全に削除・整理することができます。

本アプリケーションを用いることで、重い監視フォルダの肥大化を防ぐとともに、複数台での分散レンダリングの稼働状況をスマートに管理できます。

## 主な機能

1. **完全なフォルダ単位の自動スキャン監視**
   - 設定した監視間隔（秒）ごとに、指定した監視ルートフォルダ内の全サブフォルダをスキャンします。アプリ起動前から待機しているフォルダも即座に認識してリストに追加します。
2. **RCF ベースのステータス高精度判定**
   - AEがステータス管理に用いる標準インジケーターである `[プロジェクト名]_RCF.txt` を読み解き、`init` フラグや `(Finished` などの内部書き込みから状態を正確に取得します。
   - レポート出力（`.htm` など）が無効化されている場合でも、生成される `*_レポート.txt` からプロジェクト名や完了・エラー状態を読み取る仕組みを備えています。
3. **安全なフォルダ・ファイルのクリーンアップ（ごみ箱への削除）**
   - リストから完了したタスクを選択し「削除」すると、AEPやログだけでなく、「収集されたプロジェクトフォルダ全体」が **Windowsのごみ箱** へ移動されます。安全に容量を節約できます。
   - 他のアーカイブ用フォルダへの「移動」も可能です。
4. **ドラッグ＆ドロップによる監視登録（独自機能）**
   - AEプロジェクト（`.aep`）をアプリケーションのウィンドウにドラッグ＆ドロップするだけで、監視フォルダ内にサブディレクトリが作られ、必要な RCF ファイルとレポート用ダミーファイル（AE標準フォーマット準拠）が自動生成されます。手早く監視レンダリングキューに追加する際に便利です。

## 開発・動作環境

- **フレームワーク**: .NET 8.0 (WPF) / C#
- **アーキテクチャ**: MVVMパターン (`CommunityToolkit.Mvvm` を使用)
- **対象OS**: Windows

## 使用方法

1. アプリケーションを起動します。
2. `監視フォルダ` のテキストボックスに、After Effects側で「監視フォルダー」として指定しているルートディレクトリのパスを入力します。（例: `C:\AEWatch`）
3. `スキャン間隔(秒)` を任意の値に調整し（デフォルトは3秒）、**「監視開始」** ボタンをクリックします。
4. 監視フォルダ内に生成されたレンダリングジョブがリスト上に一覧表示されます。ステータスはリアルタイム（設定された間隔）で更新されます。
5. 完了したジョブを右クリックなどで選択し、「削除」または「移動」を実行してフォルダ整理を行います。

### D&Dによる追加
監視開始後、エクスプローラーから `.aep` ファイルを直接ウィンドウにドロップすると、即座に必要なファイル群が自動生成され、キュー（一覧）に追加されます。
